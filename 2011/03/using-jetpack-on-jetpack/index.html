    Using Jetpack on Jetpack

    March 21, 2011 by Laurian Gridinoc

    I’ve been Saturday at the State Of The Brower — it was a brilliant day, great talks and conversations and I’ve gave my longest Jetpack (Mozilla Add-​on SDK) presentation (45′) and I had no slides this time.

    I was preparing slides couple days ago then I thought that apart of duplicating bits from the existing documentation, and confuse the audience by switching between Keynote, TextMate, Terminal and Firefox; I thought that I should just show them what they will see when they will want to learn more later: the Mozilla Add-​on SDK site and online documentation, as rarely someone comes back to a set of slides.

    Now, the problem with the Jetpack documentation is that in order to get the feel of how those APIs work you have to copy paste code bits into an editor and use cfx tool to run them, or simpler, paste them into the Add-​on Builder.

    But Jetpack can modify web pages, so I wrote a Jetpack that modifies the Jetpack documentation: it inserts a “Quick Example” at the top of each module page, example that can be run right there in the browser, in that very page.

    You can install this add-​on (it is restartless, all Jetpacks are) from https://​addons​.mozilla​.org/​e​n​-​U​S​/​f​i​r​e​f​o​x​/​a​d​d​o​n​/​a​d​d​-​o​n​-​s​d​k​-​e​x​a​m​p​l​es/

    This way I was able to explain all the Jetpack modules without leaving the browser; next, I’ll see how can I embed examples from the Add-​on Builder, I’ll probably have to scrape them with the page-​worker module… (to be continued)
